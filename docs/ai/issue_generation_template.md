# Issue Generation Template

ChatGPT は repo の状態、既存 Issue/PR、関連ファイルを確認したうえで、以下の形式で Issue 文面を作成すること。

## タイトル
[TASK] 具体的な作業名

## 本文
### 概要
何をしたいかを1〜3文で記載。

### 背景
なぜ必要か、何が問題かを記載。

### 現状
関連ファイル、既存実装、既知の挙動を記載。

### 対応方針
どう直すかの方針を簡潔に記載。

### 対象範囲
- 修正対象:
- 対象外:

### 要件
- [ ] 要件1
- [ ] 要件2
- [ ] 要件3

### 完了条件
- [ ] 実装完了
- [ ] pytest 実行
- [ ] 変更ファイル一覧をPRに記載
- [ ] 影響範囲を確認
- [ ] 必要に応じて docs 更新

### 制約
- API互換維持
- 不要なリファクタ禁止
- 影響範囲最小化

### 実装メモ
必要であれば仮説・懸念点・関連Issue/PRを記載。
