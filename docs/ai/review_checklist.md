# PR Review Checklist

## 目的
このチェックリストは、ChatGPT が `tv-backtest-api` の PR をレビューする際の標準観点とする。

## 必須観点

### 1. Issue準拠
- PR が対応Issueの要件を満たしているか
- 完了条件を満たしているか
- スコープ外の変更が混ざっていないか

### 2. 破壊的変更
- API互換を壊していないか
- DBや保存形式に破壊的変更がないか
- 既存の結果ファイルや運用手順に影響がないか

### 3. 実装妥当性
- ロジックが要件に合っているか
- 仮実装やその場しのぎになっていないか
- 命名と責務分離が崩れていないか

### 4. テスト
- pytest が実行されているか
- 変更に見合うテストがあるか
- 既存機能への回帰リスクが高いのに未検証ではないか

### 5. 保守性
- 無関係な変更が混ざっていないか
- ファイル分割や責務が不自然でないか
- コメントやドキュメント追従が必要なのに漏れていないか

## 指摘の優先度
- Critical: マージ不可。バグ、破壊的変更、重大な仕様逸脱
- Major: 修正推奨。要件未達、テスト不足、保守性の大きな低下
- Minor: 任意修正。軽微な可読性、命名、コメント不足
